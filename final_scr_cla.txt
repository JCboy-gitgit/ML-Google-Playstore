Before building any model, we first cleaned the raw Google Play Store dataset because machine learning performance depends heavily on data quality. In our raw data, we found common issues: duplicated entries, missing values, inconsistent labels, and non-numeric formats.

First, we loaded the dataset and performed initial inspection using missing-value checks and data type checks.

Next, we removed duplicated apps based on app name, then reset the index so the dataset structure remained clean and consistent.

For Rating and Reviews, we handled invalid and inconsistent values:
- We identified rating outliers, especially values above 5.
- We converted Reviews into numeric format.
- If an app had zero reviews, we set rating to zero for consistency.
- Then we filled remaining missing ratings using forward fill.

For Price, we removed the dollar sign and converted the column to numeric so it could be used in computation and modeling.

For Installs, we standardized values such as 0+ and 1+, removed symbols like plus signs and commas, and converted the column into numeric values.

For Size, we standardized units by converting MB to KB and added handling for entries like "Varies with device" so they would not break numeric processing.

For Type, we applied a consistency rule based on Price:
- If Price is greater than 0, Type should be Paid.
- If Price is 0, Type should be Free.
This fixed inconsistent and missing Type values.

For Content Rating, we standardized uncommon labels, such as replacing "Adults only 18+" with "Mature 17+" and "Unrated" with "Everyone".

After that, we performed a final quality check, dropped less relevant columns such as Current Ver, Android Ver, Genres, and Last Updated, then saved the cleaned dataset.

In summary, our cleaning process reduced noise, improved consistency, and made the dataset reliable for PCA and Random Forest in the next part of our project.
